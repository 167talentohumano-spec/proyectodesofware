1. Cierre del problema y alcance del proyecto 

Descripción del problema inicial 

En la actualidad, muchos usuarios presentan dificultades para acceder a plataformas web modernas que permitan consultar contenido tecnológico de manera organizada, dinámica e intuitiva. Además, algunas aplicaciones carecen de funcionalidades interactivas como autenticación de usuarios, publicación de artículos, gestión de perfiles y administración de contenido. 

Con base en esta problemática, se planteó el desarrollo de una aplicación web tipo blog tecnológico, enfocada en la publicación y gestión de artículos relacionados con programación, desarrollo de software y tecnologías modernas. 

El sistema fue diseñado con el objetivo de ofrecer una experiencia amigable para los usuarios, permitiendo la interacción con publicaciones mediante funcionalidades modernas y una interfaz adaptable. 

 

Explicación de cómo la solución desarrollada responde al problema 

La solución implementada corresponde a una aplicación web funcional desarrollada utilizando Programación Orientada a Objetos (POO), aplicando buenas prácticas de desarrollo y arquitectura de software. 

De acuerdo con la evidencia presentada en el video, el sistema desarrollado permite: 

Registro e inicio de sesión de usuarios.  

Navegación dentro de la plataforma web.  

Publicación de artículos tecnológicos.  

Visualización de artículos publicados.  

Gestión de perfiles de usuario.  

Interacción con publicaciones mediante favoritos.  

Edición y administración de contenido.  

Diseño responsive adaptable a diferentes dispositivos.  

La plataforma implementa una interfaz moderna y organizada, facilitando el acceso a la información y mejorando la experiencia del usuario en comparación con soluciones tradicionales menos intuitivas.

3. Pipeline de CI/CD 

Introducción 

Con el objetivo de garantizar la calidad, estabilidad y automatización del proceso de desarrollo del software, se implementó un pipeline de Integración Continua y Despliegue Continuo (CI/CD) utilizando GitHub Actions. 

La implementación del pipeline permitió automatizar tareas fundamentales del proyecto, tales como: 

Construcción automática del proyecto.  

Validación de calidad del código mediante linting.  

Ejecución de pruebas automatizadas.  

Despliegue automático a producción.  

Estas automatizaciones mejoran significativamente el flujo de trabajo del equipo de desarrollo, reduciendo errores manuales y asegurando entregas más rápidas y confiables. 

Herramienta utilizada 

Para la automatización del pipeline se utilizó: 

GitHub Actions  

Repositorio del proyecto: 

Repositorio oficial del proyecto 

 

Configuración de GitHub Actions 

El pipeline fue configurado mediante un archivo YAML ubicado en: 

.github/workflows/ci-cd.yml 

La configuración permite ejecutar automáticamente el proceso CI/CD sobre cada Pull Request y también al realizar merge hacia la rama main. 

Archivo de configuración del pipeline 

ci-cd.yml 

name: Angular CI/CD Pipeline 
 
on: 
pull_request: 
 branches: 
     - main 
 
 push: 
   branches: 
     - main 
 
jobs: 
 build-test-deploy: 
   runs-on: ubuntu-latest 
 
   steps: 
 
     - name: Clonar repositorio 
       uses: actions/checkout@v4 
 
     - name: Configurar Node.js 
       uses: actions/setup-node@v4 
       with: 
         node-version: 20 
 
     - name: Instalar dependencias 
       run: npm install 
 
     - name: Ejecutar Lint 
       run: npm run lint 
 
     - name: Ejecutar pruebas unitarias 
       run: npm run test -- --watch=false --browsers=ChromeHeadless 
 
     - name: Build del proyecto 
       run: npm run build --prod 
 
     - name: Deploy a producción 
       if: github.ref == 'refs/heads/main' 
       run: echo "Deploy realizado correctamente"
